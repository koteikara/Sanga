# Googleスプレッドシート実運用テンプレート

この文書は、京都サンガF.C.年間スケジュールをGoogleスプレッドシートで一元管理し、CSV出力後に `matches.json` 形式へ変換するための実運用テンプレートです。現段階ではGoogle Sheets API連携、自動取得、認証情報の作成は行わず、人が入力・確認しやすいシート構成を整えます。

## 1. スプレッドシートの目的

* 京都サンガF.C.の年間スケジュールを、1つのGoogleスプレッドシートで管理します。
* 公開用JSON `public/data/matches.json` に変換できるCSVを、人が安全に出力できる状態にします。
* 日程、候補日、状態、根拠URL、注記を分けて管理し、未確定情報と確定情報を区別します。
* 将来Google Sheets API連携を追加する場合にも、列名と運用ルールをそのまま使える形にします。

## 2. 作成するシート構成

| シート名 | 用途 | CSV出力対象 |
| --- | --- | --- |
| 試合 | `matches.json` に変換する本体データを管理する | 対象 |
| 注記 | `※1` 〜 `※7` などの注記番号と本文を管理する | 原則対象外 |
| 設定 | シーズン、チーム名、競技名、根拠URL、更新日時などの共通メタ情報を管理する | 原則対象外 |
| 確認 | 入力漏れ、ID重複、日付形式、候補日、状態値などの確認観点を整理する | 対象外 |

CSVとして書き出すのは、原則として `試合` シートだけです。ほかのシートは入力補助、確認、将来拡張のために使います。

## 3. 各シートの役割

### 試合

`matches.json` に変換する本体データを管理するシートです。1行を1試合として入力します。列名は日本語を基本にし、CSV生成スクリプト `tools/generate-matches-from-csv.js` が扱える列を含めます。

### 注記

`※1` 〜 `※7` など、公開画面に表示してよい注記本文を管理するシートです。将来的に `試合` シートの `注記番号` から参照しやすいよう、番号と本文を分けて管理します。

推奨列:

| 列名 | 内容 | 入力例 |
| --- | --- | --- |
| 注記番号 | 注記を識別する番号 | ※1 |
| 注記本文 | 公開してよい本文 | 26/12/16(水)もしくは27/4/14(水)のいずれかで開催予定です。 |
| 根拠URL | 注記の根拠となる公開URL | https://www.sanga-fc.jp/ |
| 確認日 | 根拠URLを確認した日 | 2026-06-18 |
| メモ | 非公開の運用メモ | 公式発表後に更新する |

### 設定

シーズン、チーム名、競技名、根拠URL、更新日時など、複数行に共通するメタ情報を管理するシートです。現時点の生成スクリプトでは既定値を使いますが、将来の自動化に備えて値を整理します。

推奨列:

| 項目 | 値の例 | 用途 |
| --- | --- | --- |
| season | 2026-27 | JSONの `meta.season` や各試合の `season` |
| team | 京都サンガF.C. | JSONの `meta.team` |
| competition | J1 | JSONの `competition` |
| competition_label | 明治安田J1リーグ | JSONの `competition_label` |
| default_source_url | https://www.sanga-fc.jp/ | 根拠URL未入力時の確認先メモ |
| updated_at | 2026-06-18 10:00 | 最終確認日時のメモ |

### 確認

入力内容を人が確認するためのメモ用シートです。自動関数の実装は必須ではありません。まずは確認観点と結果を残せるようにします。

確認観点:

* IDが `sec01` 〜 `sec38` の形式になっているか。
* IDの重複がないか。
* 表示順が1〜38で重複なく並んでいるか。
* `開催日` が `YYYY-MM-DD` 形式になっているか。
* `候補日` が `|` 区切りで、各日付が `YYYY-MM-DD` 形式になっているか。
* `開催日` と `候補日` が同時入力されていないか。
* `状態` が `確定` または `未確定` になっているか。
* `ホームアウェイ` が `H` または `A` になっているか。
* `ホームアウェイ表示` が `ホーム` または `アウェイ` になっているか。
* `根拠URL` に、可能な限り公式サイトやJリーグ公式サイトなどの公開URLが入っているか。
* 公開してはいけない内部メモが `注記` に入っていないか。
* `メモ` がCSV変換対象外の運用メモとして扱われているか。

## 4. 入力用シートの列順

`試合` シートでは、次の列順を使います。ヘッダー行だけを作る場合は `docs/sheets/schedule.template.csv` をコピーして貼り付けます。

```text
ID,表示順,節,開催年,開催日,候補日,状態,ホームアウェイ,ホームアウェイ表示,対戦相手,対戦相手コード,会場,注記番号,注記,根拠URL,確認日,メモ
```

### CSV変換に必要な列

| 列名 | 用途 | 入力例 |
| --- | --- | --- |
| ID | JSONの `id` | sec01 |
| 表示順 | 並び順 | 1 |
| 節 | JSONの `round` | 第1節 |
| 開催年 | シーズン内の開催年 | 2026 |
| 開催日 | JSONの `match_date` | 2026-09-02 |
| 候補日 | JSONの `date_candidates` | 2026-12-16\|2027-04-14 |
| 状態 | JSONの `status` / `status_label` に変換 | 確定 |
| ホームアウェイ | JSONの `home_away` | H |
| ホームアウェイ表示 | JSONの `home_away_label` | ホーム |
| 対戦相手 | JSONの `opponent` | 神戸 |
| 対戦相手コード | JSONの `opponent_code` | vissel |
| 会場 | JSONの `venue` | サンガスタジアム |
| 注記 | JSONの `note` | ※1: 開催候補日あり |
| 根拠URL | JSONの `source_url` | https://www.sanga-fc.jp/ |

### 運用確認用の列

| 列名 | 用途 | CSV変換での扱い |
| --- | --- | --- |
| 注記番号 | `注記` シートと照合するための番号 | 現時点では直接変換しない |
| 確認日 | 根拠URLを確認した日 | 現時点では直接変換しない |
| メモ | 非公開の運用メモ | 変換対象外 |

`注記番号` は運用しやすさのために追加します。最終的にCSVへ出力する `注記` は、`※1: ...` のように番号と本文が分かる形式にします。

## 5. 入力ルール

* `ID` は `sec01` 〜 `sec38` の形式で入力します。
* `表示順` は1〜38の整数で入力します。
* `開催日` は `YYYY-MM-DD` 形式で入力します。
* `候補日` は複数ある場合、`|` 区切りで入力します。
* `開催日` と `候補日` は同時に入力しません。
* 未定の場合は `開催日` と `候補日` を空欄にし、`状態` を `未確定` にします。
* 確定している場合は `状態` を `確定` にします。
* `ホームアウェイ` は `H` または `A` を入力します。
* `根拠URL` は可能な限り公式URLを入力します。
* 内部メモや非公開情報は `注記` に入れません。
* `メモ` はCSV変換対象外として扱います。

## 8. 初期データ作成手順

38件分の試合行を最初から手入力せず、現在の正本である `public/data/matches.json` から、Googleスプレッドシートの `試合` シートへ取り込む初期CSVを作成できます。生成済みの初期取り込み用CSVとして `docs/sheets/schedule.initial.csv` を利用できます。

```bash
node tools/export-matches-to-sheet-csv.js public/data/matches.json tmp/schedule-from-current-json.csv
```

初期データ作成から検証までの流れは次のとおりです。

1. `public/data/matches.json` から `tmp/schedule-from-current-json.csv` を生成します。
2. 検証に成功したCSVを `docs/sheets/schedule.initial.csv` として保存します。
3. Googleスプレッドシートの「ファイル」→「インポート」→「アップロード」から `docs/sheets/schedule.initial.csv` を取り込みます。
4. 取り込み先は「新しいシートを挿入」を選ぶか、既存の「試合」シートへ貼り付けます。
5. 取り込み後、シート名を「試合」にします。
6. ヘッダー1行 + 試合38行があることを確認します。
7. プルダウンや確認シートを設定します。
8. スプレッドシート上で38件を確認・修正します。
9. CSVでダウンロードします。
10. `generate` → `validate` を実行します。

### Googleスプレッドシートへの初期CSV取り込み手順

1. Googleスプレッドシートを新規作成します。
2. ファイル → インポート → アップロード を選びます。
3. `docs/sheets/schedule.initial.csv` をアップロードします。
4. 取り込み方法は「新しいシートを挿入」を選びます。
5. 取り込まれたシート名を「試合」に変更します。
6. 行数がヘッダー1行 + 試合38行になっていることを確認します。
7. 「注記」「設定」「確認」シートを追加します。
8. 状態、ホームアウェイ、ホームアウェイ表示、開催年、注記番号、対戦相手コードにプルダウンを設定します。

`tmp/` 配下は一時作業用なのでコミットしません。スプレッドシート上で修正した内容を公開用JSONへ反映する場合は、必ずCSVをダウンロードしてから `tools/generate-matches-from-csv.js` と `tools/validate-generated-matches.js` で確認します。

## 9. CSV出力手順

1. Googleスプレッドシートで `試合` シートを開きます。
2. フィルタで一部行だけが表示されていないか確認します。
3. ヘッダー行と入力済みの試合行がすべて表示されていることを確認します。
4. メニューから「ファイル」→「ダウンロード」→「カンマ区切り形式（.csv）」を選択します。
5. ダウンロードしたCSVをリポジトリ外で確認し、必要に応じて `tmp/` や作業用ファイルとして配置します。
6. 本番用 `public/data/matches.json` は、生成結果の検証が終わるまで直接上書きしません。

## 10. CSV出力後の生成・検証手順

CSV出力後は、確認用JSONを生成してから検証します。Googleスプレッドシートから出力したCSVは `tmp/schedule-from-sheet.csv` などに置き、生成結果は `tmp/matches.generated.json` に出力します。`tmp/` 配下は一時ファイルなのでコミットしません。サンプルとして残すCSVは `docs/sheets/schedule.sample.csv`、テンプレートとして残すCSVは `docs/sheets/schedule.template.csv` です。

```bash
node tools/generate-matches-from-csv.js docs/sheets/schedule.sample.csv tmp/matches.generated.json
node tools/validate-generated-matches.js tmp/matches.generated.json
```

本番用38件として確認する場合は、件数チェックも指定します。

```bash
node tools/validate-generated-matches.js tmp/matches.generated.json --expected-count 38
```

公開用JSONへ反映した後は、従来の検証も実行します。

```bash
node tools/validate-matches.js
node tools/export-matches-review.js
```

## 11. よくある入力ミスと対処

| 入力ミス | 起きる問題 | 対処 |
| --- | --- | --- |
| `ID` が `sec1` になっている | ID形式が揃わず確認しにくい | `sec01` のように2桁へ直す |
| `表示順` が空欄または重複 | 並び順が意図と変わる | 1〜38を重複なく入力する |
| `開催日` が `2026/09/02` になっている | 日付形式エラーになる | `2026-09-02` に直す |
| `開催日` と `候補日` の両方が入っている | 生成・検証でエラーになる | 確定日がある場合は `候補日` を空欄にする |
| `候補日` を読点や改行で区切っている | 候補日として分割できない | `2026-12-16|2027-04-14` のように `|` 区切りにする |
| `状態` が `確認中` などになっている | 現行スクリプトの許容値外になる | `確定` または `未確定` に直す |
| `ホームアウェイ` が `ホーム` になっている | 現行スクリプトの許容値外になる | `H` または `A` に直す |
| 非公開メモを `注記` に入れている | 公開JSONへ出る可能性がある | `メモ` に移し、`注記` は公開してよい文だけにする |
| `根拠URL` が空欄 | 後で確認しにくい | 可能な限り公式URLを入力する |

## 12. 本番反映までの流れ

1. Googleスプレッドシートの `試合` シートへ入力します。
2. `注記`、`設定`、`確認` シートで注記本文、共通情報、確認観点を確認します。
3. `試合` シートをCSV形式でダウンロードします。
4. `tools/generate-matches-from-csv.js` で `tmp/matches.generated.json` を生成します。
5. `tools/validate-generated-matches.js` で生成JSONを検証します。
6. 生成JSONの件数、ID、日付、候補日、状態、注記、根拠URLを目視確認します。
7. 問題がない場合だけ、手作業で `public/data/matches.json` へ反映します。
8. `tools/validate-matches.js` と `tools/export-matches-review.js` で公開用JSONを確認します。
9. GitHub Pagesなどの確認環境で表示を確認します。
10. 本番サーバーへのアップロードは、明示的な反映指示がある場合だけ実行します。

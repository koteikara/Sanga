# Googleスプレッドシート列定義

この文書は、Googleスプレッドシートで管理する年間スケジュールページ用の「試合」シート列と、公開用JSON `public/data/matches.json` の項目対応を整理するものです。現段階ではGoogle Sheets API連携は行わず、スプレッドシートからCSVを書き出し、`tools/generate-matches-from-csv.js` で確認用JSONを生成する前提です。

## 基本方針

* 列名は日本語を基本にし、運用者が入力しやすい表現にします。
* JSON生成時に、日本語列名を `matches.json` の英語キーへ変換します。
* 生成先はまず `tmp/matches.generated.json` などの確認用ファイルにし、本番用 `public/data/matches.json` は直接上書きしません。
* 公式情報に基づく日程は、根拠URLを必ず残します。
* 未確定情報は、確定日と区別して `状態`、`候補日`、`公開メモ`、`注記` で扱います。
* 年間スケジュールページの「試合」シートには、試合そのものの表示・絞り込み・共有・戦績更新に必要な項目を置きます。
* チケット販売開始日時、ファンクラブグレード別販売開始日時、イベント申込開始・終了日時、グッズ販売開始日時、キャンペーン締切などは、ニュース連動カレンダー側または別の日程イベントデータで扱います。
* 個人メモ、参戦予定、視聴予定、気になる試合、観戦準備メモ、チケット購入済みなど、閲覧者ごとに変わる情報はLocalStorageで扱い、スプレッドシートには入れません。

## 推奨列順

「試合」シートでは、今後の機能拡張を見据えて次の列順を推奨します。

```text
ID,表示順,大会,大会表示,節,開催年,開催日,候補日,キックオフ時刻,状態,試合状態,ホームアウェイ,ホームアウェイ表示,対戦相手,対戦相手コード,会場,京都得点,相手得点,結果,チケットURL,イベントURL,放送配信,放送配信URL,注記番号,注記,公開メモ,SNS用短縮タイトル,表示フラグ,根拠URL,確認日,メモ
```

`docs/sheets/schedule.initial.csv` は現行JSONから作成した初期取り込み用CSVであり、この完成形の列をすべて含んでいるとは限りません。スプレッドシートを本格運用する前に、上記の拡張列を追加します。

## 既存列と追加列の意味の違い

### `状態`

`状態` は、日程の確定状態を表す列です。開催日、候補日、会場などの情報が確定しているかを管理します。

例:

```text
確定
未確定
```

JSONでは `status` / `status_label` に変換します。`確定` は `confirmed`、`未確定` は `tentative` として扱う方針です。

### `試合状態`

`試合状態` は、試合そのものの進行状態を表す列です。日程が確定しているかではなく、試合前か、試合中か、試合終了後か、延期・中止になったかを管理します。

例:

```text
試合前
試合中
試合終了
延期
中止
未定
```

たとえば、開催日が確定している試合でも、試合前であれば `状態 = 確定`、`試合状態 = 試合前` になります。延期が発表された場合は、日程の再確定状況に応じて `状態` を見直しつつ、`試合状態 = 延期` として表します。

### `公開メモ`

`公開メモ` は、公開画面に表示してよい短い補足です。試合カード内に短く出す想定で、閲覧者にそのまま見えてよい内容だけを入力します。

例:

```text
会場未定
日程変更の可能性あり
チケット情報確認中
```

長い注記や注記番号付きの補足は `注記` を使い、短いカード補助表示は `公開メモ` を使います。

### `メモ`

`メモ` は、運用者向けの非公開メモです。公開JSONや画面表示には原則出しません。

例:

```text
公式発表待ち
次回更新時に確認
```

`メモ` には公開できない確認状況や作業メモを書けますが、CSV変換・公開JSON出力の対象外として扱います。

### 個人メモ

閲覧者ごとの個人メモは、スプレッドシートではなくLocalStorageで扱います。個人メモ、参戦予定、視聴予定、気になる試合、観戦準備メモ、チケット購入済みなどは閲覧者ごとに内容が異なるため、全員共通のスプレッドシートには保存しません。

`公開メモ` は全閲覧者に表示してよい共通メモ、`メモ` は運用者だけが見る非公開メモ、個人メモは閲覧者の端末に保存するLocalStorageデータ、という役割で分けます。

## 生成時に補完するJSON項目

現行の `tools/generate-matches-from-csv.js` は、まだ拡張列すべてを変換しません。現時点でCSVに列を持たない、または変換対象外の項目は、生成スクリプト側で次のように補完しています。

| JSONキー | 現行の補完方針 |
| --- | --- |
| season | 既定値 `2026-27` |
| competition | 既定値 `J1` |
| competition_label | 既定値 `明治安田J1リーグ` |
| kickoff_time | 空文字 |
| result | 空文字 |
| ticket_url | 空文字 |
| broadcast | 空文字 |
| source_checked_at | 生成日時 |
| updated_at | 生成日時 |
| meta | `season`、`team`、`updated_at`、`source` を出力 |

今後、拡張列をCSV生成・検証スクリプトへ反映する場合は、この文書のJSONキー候補に合わせて段階的に対応します。

## 日程状態の扱い

### 日付確定

* `開催日` に `YYYY-MM-DD` を入力します。
* `候補日` は空欄にします。
* `状態` は `確定` にします。
* `試合状態` は原則 `試合前` にします。
* JSONでは `match_date` に日付を入れ、`date_candidates` は空配列にします。

### 日付未定

* `開催日` と `候補日` を空欄にします。
* `状態` は `未確定` にします。
* 必要に応じて `公開メモ` に「会場未定」「日程変更の可能性あり」などを入力します。
* JSONでは `match_date` を空文字、`date_candidates` を空配列にします。

### 候補日がある日程

* `開催日` は空欄にします。
* `候補日` に候補日を `|` 区切りで入力します。
* `状態` は原則 `未確定` にします。
* JSONでは `date_candidates` に候補日配列を出力します。

### 注記付き日程

* 画面に表示してよい補足のみ `注記` または `公開メモ` に入力します。
* 注記番号を出す場合は `※1:` のように、番号を判別できる形式にします。
* 内部メモや非公開情報は `注記`、`公開メモ` に入れません。

## 生成後の検証手順

スプレッドシートからCSVを書き出した後は、生成結果をすぐに本番用JSONへ反映せず、次の順序で確認します。

1. GoogleスプレッドシートからCSVを出力します。
2. CSVから確認用JSON `tmp/matches.generated.json` を生成します。

   ```bash
   node tools/generate-matches-from-csv.js docs/sheets/schedule.sample.csv tmp/matches.generated.json
   ```

3. 生成したJSONを `tools/validate-generated-matches.js` で検証します。

   ```bash
   node tools/validate-generated-matches.js tmp/matches.generated.json
   ```

4. 本番用49件として運用するデータでは、件数チェックも指定します。

   ```bash
   node tools/validate-generated-matches.js tmp/matches.generated.json --expected-count 49
   ```

5. エラーがなく、内容を確認して問題がなければ `public/data/matches.json` へ反映します。
6. 反映後に、公開用JSON向けの従来検証を実行します。

   ```bash
   node tools/validate-matches.js
   ```

7. GitHub Pagesの確認環境で表示と操作に問題がないことを確認します。
8. 本番サーバーへのデプロイは、明示的に反映する段階で実行します。

## 注意事項

* サービスアカウントJSON、APIキー、`.env` などの認証情報は作成しません。
* CSVは確認用JSON生成の入力であり、生成結果を確認してから `public/data/matches.json` へ反映します。
* `根拠URL` には可能な限り公式サイト、Jリーグ公式サイトなどの公開URLを記録します。
* `docs/sheets/schedule.initial.csv` は現行JSON由来の初期取り込み用として残し、今回の拡張列の完成形とは分けて扱います。

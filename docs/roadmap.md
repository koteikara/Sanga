# 開発ロードマップ

## 目的

このドキュメントは、京都サンガF.C.年間スケジュールページの今後の開発予定、完了済み機能、次に取り組む候補、保留事項を一箇所に整理するためのロードマップです。

詳細な設計や運用手順は既存ドキュメントに分散しているため、このドキュメントでは全体像と優先順位を確認できるようにします。

## 現在の到達点

現時点では、年間スケジュールページの基本機能は `public/data/matches.json` を中心に動作する構成へ移行済みです。表示列切替、表示モード切替、フィルタリング、色枠状態のLocalStorage保存、設定パネルへの表示関連UI集約、使い方パネル、LocalStorage削除、共有用画像生成、共有画像の保存導線、作成者表記、免責事項は実装済みまたは本番反映済みとして扱います。

運用面では、GitHub Pages確認、本番サーバーへの手動デプロイ、スプレッドシート更新から本番反映までの流れ、UIプロトタイプ運用ルール、CSS棚卸し・確認観点整理も進んでいます。一方で、CSS整理やJavaScript整理、Googleスプレッドシート運用、実ブラウザ確認の自動化などは、継続確認が必要な領域として扱います。

## 開発方針

- 年間スケジュールページは、試合予定の確認、パーソナライズ、SNS共有に集中する。
- ニュース由来のチケット販売開始、イベント申込締切などはニュース連動カレンダー側を基本にする。
- UIを伴う大きな変更は、先に `experiments/` でプロトタイプを作る。
- 個人情報・個人状態はLocalStorage中心で扱う。
- 公開JSONには個人メモや個人状態を含めない。
- 本番ページから検証用プロトタイプへリンクしない。
- 日程表示欄はできるだけごちゃつかせない。
- 表示関連UIは設定パネルへ集約する。

## 保留・対象外

| 項目 | 状態 | 理由 |
| --- | --- | --- |
| SNS投稿文生成 | 対象外 | 画像生成でSNS補助として十分と判断したため、現時点では対象外とする。投稿文は利用者ごとの意図が異なり、UIが複雑化するため、実装しない。 |

| ログイン機能 | 対象外 | 個人情報管理や認証基盤が必要になるため、LocalStorage中心の方針から外れる。 |
| 外部API連携を伴う個人機能 | 対象外 | 認証情報、個人情報、運用負荷が増えるため、現時点では扱わない。 |

## 2026-06-24 JavaScript整理の進捗

表示列・表示モード関連処理は、定義、正規化、CSSクラス適用、ボタン状態更新の責務を分ける整理を一部実施済みです。既存のLocalStorageキーと保存形式、表示列 `1 / 2 / 3 / 4`、表示モード `card / compact`、共有画像生成へのDOM状態反映は維持しています。

今後の継続候補は、LocalStorage処理全体の見通し改善、試合カード描画処理、共有画像生成処理、使い方ダイアログ・設定パネル操作の整理です。これらは表示仕様やアクセシビリティへの影響が大きいため、別PRで確認・検証を挟みながら進めます。

## 2026-06-24 JavaScript整理の進捗: LocalStorage関連

LocalStorage関連処理は、キー一覧、汎用読み書き、JSON保存・復元、保存内容削除、削除後のUI初期化の責務分離を一部実施済みです。既存キーと保存形式は維持し、新しいLocalStorageキーは追加していません。

引き続き、試合カード描画、共有画像生成、使い方ダイアログ・設定パネル操作の整理は未着手または継続候補として扱います。これらはDOM構造、表示仕様、アクセシビリティへの影響が大きいため、別PRで確認・検証を挟みながら進めます。

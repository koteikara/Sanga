# データ定義

このドキュメントは、京都サンガF.C.年間スケジュールサイトで使用する日程データの定義をまとめるものです。

本プロジェクトでは、Googleスプレッドシートを日程データの正本とし、そこから公開用JSONまたはHTMLを生成することを目指します。

## 基本方針

* スプレッドシートの列名は、日本語で分かりやすくする
* JSONやJavaScriptなどの内部キーは、英語のキー名を使用する
* スプレッドシートでは、人が見て入力しやすいことを優先する
* 自動処理では、内部キーに変換して扱う
* 公開サイトのHTMLには、可能な限り日程データを直接書き込まない
* 情報元URLと確認日時を記録する
* 未確定情報は、確定情報と区別して扱う

## データ管理の考え方

データの流れは、次の形を想定します。

```text
Googleスプレッドシート
  ↓
日本語列名を内部キーへ変換
  ↓
公開用JSONを生成
  ↓
公開サイトで読み込み・表示
```

スプレッドシート上では、次のように日本語の列名を使います。

```text
ID｜シーズン｜大会｜節・回戦｜試合日｜開始時刻｜ホーム/アウェイ｜対戦相手｜会場｜結果｜チケットURL｜放送・配信｜情報元URL｜情報確認日時｜状態｜備考｜更新日時
```

JSONでは、次のような英語キーに変換します。

```json
{
  "id": "2026-j1-01",
  "season": "2026-27",
  "competition": "J1",
  "round": "第1節",
  "match_date": "2026-02-14",
  "kickoff_time": "14:00",
  "home_away": "H",
  "opponent": "清水エスパルス",
  "venue": "サンガスタジアム by KYOCERA",
  "result": "",
  "ticket_url": "",
  "broadcast": "DAZN",
  "source_url": "",
  "source_checked_at": "2026-06-18 10:00",
  "status": "confirmed",
  "note": "",
  "updated_at": "2026-06-18 10:00"
}
```

## メインシート: 試合日程

シート名は、分かりやすさを優先して `試合日程` とします。

| スプレッドシート列名 | 内部キー              | 必須 | 内容              | 入力例                  |
| ---------- | ----------------- | -: | --------------- | -------------------- |
| ID         | id                |  ○ | 試合・予定を一意に識別するID | 2026-j1-01           |
| シーズン       | season            |  ○ | 対象シーズン          | 2026-27              |
| 大会         | competition       |  ○ | 大会区分            | J1                   |
| 節・回戦       | round             |    | 節、回戦、ステージなど     | 第1節                  |
| 試合日        | match_date        |    | 試合日             | 2026-02-14           |
| 開始時刻       | kickoff_time      |    | キックオフ時刻         | 14:00                |
| ホーム/アウェイ   | home_away         |    | ホーム、アウェイ、中立地など  | H                    |
| 対戦相手       | opponent          |    | 対戦相手            | 清水エスパルス              |
| 会場         | venue             |    | 会場名             | サンガスタジアム by KYOCERA  |
| 結果         | result            |    | 試合結果            | 2-1                  |
| チケットURL    | ticket_url        |    | チケット情報ページのURL   | https://example.com/ |
| 放送・配信      | broadcast         |    | 放送・配信情報         | DAZN                 |
| 情報元URL     | source_url        |    | 情報の出典URL        | https://example.com/ |
| 情報確認日時     | source_checked_at |    | 情報を確認した日時       | 2026-06-18 10:00     |
| 状態         | status            |  ○ | 確定、未確定、延期、中止など  | confirmed            |
| 備考         | note              |    | 表示用または管理用の補足    | 開催日未定                |
| 更新日時       | updated_at        |    | データを更新した日時      | 2026-06-18 10:00     |

## IDの命名ルール

`ID` は、試合や予定を一意に識別するために使用します。

基本形は次の形式とします。

```text
年度-大会-番号
```

例:

| 内容                      | ID例             |
| ----------------------- | --------------- |
| 2026-27シーズン J1 第1節      | 2026-j1-01      |
| 2026-27シーズン J1 第2節      | 2026-j1-02      |
| 2026-27シーズン ルヴァンカップ 第1戦 | 2026-levain-01  |
| 2026-27シーズン 天皇杯 3回戦     | 2026-emperor-03 |

試合以外の予定を扱う場合は、次のような形式を検討します。

```text
2026-event-001
2026-ticket-001
```

## ホーム/アウェイの値

スプレッドシートの `ホーム/アウェイ` 列には、次のいずれかを入力します。

| 入力値 | 内部値 | 意味   |
| --- | --- | ---- |
| H   | H   | ホーム  |
| A   | A   | アウェイ |
| N   | N   | 中立地  |
| 未定  | TBD | 未定   |

表示上は、必要に応じて次のように変換します。

| 内部値 | 表示例  |
| --- | ---- |
| H   | ホーム  |
| A   | アウェイ |
| N   | 中立地  |
| TBD | 未定   |

## 状態の値

スプレッドシートの `状態` 列には、次のいずれかを入力します。

| 入力値 | 内部値       | 意味           |
| --- | --------- | ------------ |
| 確定  | confirmed | 日程・内容が確定している |
| 未確定 | tentative | 一部情報が未確定     |
| 延期  | postponed | 延期           |
| 中止  | canceled  | 中止           |
| 非表示 | hidden    | 公開サイトには表示しない |
| 確認中 | checking  | 情報確認中        |

公開サイトでの表示は、内部値に応じて制御します。

| 内部値       | 公開サイトでの扱い              |
| --------- | ---------------------- |
| confirmed | 通常表示                   |
| tentative | 未確定であることが分かるように表示      |
| postponed | 延期として表示                |
| canceled  | 中止として表示                |
| hidden    | 表示しない                  |
| checking  | 原則として表示しない、または確認中として表示 |

## 日付・時刻の入力ルール

### 試合日

`試合日` は次の形式で入力します。

```text
YYYY-MM-DD
```

例:

```text
2026-02-14
```

未定の場合は空欄にします。

### 開始時刻

`開始時刻` は次の形式で入力します。

```text
HH:MM
```

例:

```text
14:00
19:30
```

未定の場合は空欄にします。

### 情報確認日時・更新日時

`情報確認日時` と `更新日時` は、次の形式を基本とします。

```text
YYYY-MM-DD HH:MM
```

例:

```text
2026-06-18 10:00
```

## 入力時の注意

* 1行につき1試合または1予定を入力する
* 日付は `YYYY-MM-DD` で統一する
* 時刻は `HH:MM` で統一する
* 未定の項目は無理に入力せず空欄にする
* 出典がある場合は `情報元URL` を入力する
* 情報を確認したら `情報確認日時` を更新する
* 公開したくない行は `状態` を `非表示` にする
* 備考には、画面に表示してよい内容と内部メモが混ざらないよう注意する

## 備考欄の扱い

`備考` は公開サイトに表示される可能性がある項目です。

内部メモとして使いたい内容が増える場合は、将来的に次の列を分けることを検討します。

| 列名   | 用途             |
| ---- | -------------- |
| 表示備考 | 公開サイトに表示してよい補足 |
| 管理メモ | 公開しない内部メモ      |

初期段階では `備考` のみで運用し、必要になった段階で分割します。
